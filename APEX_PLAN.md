## 1. Goal

Build APEX: a mobile-first Next.js 14 web app that analyzes a user's face via
the Anthropic API and produces a brutally honest, evidence-based 90-day physical
optimization roadmap, exportable to PDF.

### 4.2 Shared lib
6. `lib/utils.ts`: shadcn `cn()` helper (clsx + tailwind-merge).
7. `lib/types.ts`: TypeScript types ŌĆö `UserProfile` (age, heightCm, weightKg,
   hasBeard, doesSkincare, sleepHours, dietType, trainingFrequency),
   `CategoryScore` (name, score, assessment, highest_leverage_action),
   `FaceAnalysis` (overall_score, categories[], strongest_feature,
   highest_leverage_change, notes), `Roadmap` (phases with weeks + tasks),
   `FlowStep` enum (`upload | analyzing | scores | deepdive | roadmap | export`).
8. `lib/ollama.ts`: export `OLLAMA_URL = "http://localhost:11434"` and
   `MODEL = "llava"` as constants. Export three typed helpers ŌĆö all use plain
   `fetch`, no SDK:
   - `ollamaGenerate(prompt, images?: string[])` ŌåÆ POST `/api/generate` with
     `{ model, prompt, images, stream: false, format: "json" }`. Returns the
     `response` string. Use for analyze and roadmap (JSON output).
   - `ollamaGenerateStream(prompt)` ŌåÆ POST `/api/generate` with
     `{ model, prompt, stream: true }`. Returns an `AsyncGenerator<string>`
     that yields each token. Use for deep-dive (streamed prose).
   - `safeParseJSON<T>(raw)` ŌåÆ strips ```` ```json ```` fences, attempts
     `JSON.parse`, returns `{ ok, data, error }`.
   On fetch error (ECONNREFUSED), throw with message:
   "Ollama is not running. Start it with: ollama serve"
9. `lib/prompts.ts`: export the three system prompts VERBATIM as written in
   ┬¦6 of this document ŌĆö `FACE_ANALYSIS_PROMPT` (constant string),
   `deepDivePrompt(category, score)` and `roadmapPrompt(analysisJSON, profile)`
   (functions that interpolate). These strings are passed to Gemini as
   `systemInstruction`.
10. `lib/image.ts`: client-side helper `compressImage(file)` ŌĆö draw to canvas,
    resize so the long edge Ōēż 1568px, export JPEG quality 0.85, return
    `{ base64, mediaType }` (base64 without the data-URL prefix).

### 4.3 API routes (server-side, `runtime = "nodejs"`)
All routes call Ollama via the helpers in `lib/ollama.ts`. No API key, no SDK.
Add `export const runtime = "nodejs"` and `export const maxDuration = 120` to
every route file (local Ollama can be slow on first inference).

11. `POST /api/analyze`: body = `{ front: {base64,mediaType}, side?: {...},
    profile }`. Build a single prompt string that combines `FACE_ANALYSIS_PROMPT`
    + the profile summary + instruction to return ONLY valid JSON. Pass
    `front.base64` (and `side.base64` if present) as the `images` array to
    `ollamaGenerate(prompt, images)`. Run the response through
    `safeParseJSON<FaceAnalysis>`. On parse failure, retry ONCE with an
    appended "return ONLY valid JSON, no prose" line. Return the parsed
    analysis or a 502 with a readable message. Do NOT stream.
12. `POST /api/deep-dive`: body = `{ category, score }`. Build the prompt from
    `deepDivePrompt(category, score)` (see ┬¦6 ŌĆö adapted for local knowledge,
    no web search). Use `ollamaGenerateStream(prompt)` and pipe each yielded
    token to the client as a `text/plain` streaming `Response` via a
    `ReadableStream`. Deep dive returns prose, not JSON. No images needed.
13. `POST /api/roadmap`: body = `{ analysis, profile }`. Build the prompt from
    `roadmapPrompt(...)`. Call `ollamaGenerate(prompt)` (no images). Parse with
    `safeParseJSON<Roadmap>` (+ one retry). Return the roadmap JSON. Do NOT
    stream.
14. Every route: wrap in try/catch. If `ollamaGenerate` throws "Ollama is not
    running", return a 503 with that exact message so the UI can show a clear
    "Please start Ollama" error instead of a blank crash.

### 4.4 State machine
15. `hooks/useApexFlow.ts`: a `useReducer`-based hook holding `step`,
    `profile`, `frontPhoto`, `sidePhoto`, `analysis`, `activeCategory`,
    `roadmap`, and per-request `loading`/`error`. Exposes actions: `setPhotos`,
    `setProfile`, `runAnalysis`, `openDeepDive(category)`, `closeDeepDive`,
    `runRoadmap`, `goToExport`, `reset`. Persist `analysis` + `roadmap` +
    `profile` to `sessionStorage` so a refresh mid-flow does not lose work.
    **CRITICAL ŌĆö SSR guard**: all `sessionStorage` reads/writes must be wrapped
    in `if (typeof window !== 'undefined')` because Next.js executes hooks
    server-side during hydration and `sessionStorage` does not exist there.
    Use a `useEffect` to rehydrate from sessionStorage after mount, never in
    the reducer initializer.

## 5. Edge cases
- Claude returns markdown-fenced or trailing-prose JSON ŌåÆ `safeParseJSON`
  strips fences; one automatic retry; then a user-facing error with a Retry
  button. Never crash on `JSON.parse`.
- Ollama not running ŌåÆ routes return 503 with "Please start Ollama: run
  `ollama serve` in your terminal, then retry"; UI shows this message clearly.
- Large/HEIC photos ŌåÆ `compressImage` resizes; reject non-image files in the
  picker.
- Side photo omitted ŌåÆ analysis proceeds with the front photo only; prompt
  text notes the side photo is absent.
- Ollama slow on first inference (model cold-start) ŌåÆ `maxDuration = 120`;
  show LoadingState with honest messages so the user knows it's working.
- Deep-dive stream interrupted ŌåÆ show partial content + a Retry control.
- PDF with a long roadmap ŌåÆ multi-page; verify nothing is cut off.
- Every screen must look correct at 375px width and at desktop widths.

## 6. System prompts ŌĆö embed VERBATIM in `lib/prompts.ts`

**FACE_ANALYSIS_PROMPT** (static system string):

```
You are a world-class facial aesthetics analyst trained on the latest research in orthotropics, maxillofacial science, evolutionary biology, and cosmetic medicine. Your job is to give an honest, specific, evidence-based facial assessment. Do NOT give empty compliments. Be direct but constructive.

Analyze the uploaded face across these exact categories:

1. FACIAL SYMMETRY ŌĆö Compare left vs right side proportions. Note any significant asymmetries in eye level, jaw angle, or brow position.

2. JAWLINE & MANDIBULAR DEFINITION ŌĆö Assess jaw angle, gonial angle visibility, chin projection, and overall lower third strength. Reference ideal proportions from orthotropic research (Mew, AGGA studies).

3. EYE AREA ŌĆö Evaluate canthal tilt (positive vs negative), orbital bone projection, under-eye hollowing, brow positioning, and interocular distance.

4. NOSE & MIDFACE ŌĆö Assess nose-to-face ratio, tip projection, nasolabial angle. Evaluate midface length and cheekbone projection.

5. SKIN QUALITY ŌĆö Note visible texture, pores, pigmentation, acne scarring, overall clarity. Be specific about what you observe.

6. FACIAL FAT DISTRIBUTION ŌĆö Assess where fat is sitting (cheeks, under chin, neck). Estimate whether this is masking underlying bone structure.

7. OVERALL FACIAL HARMONY ŌĆö Golden ratio alignment, thirds proportions (hairline to brow, brow to nose, nose to chin), and overall aesthetic balance.

8. STRONGEST FEATURE ŌĆö What is genuinely working well.

9. HIGHEST LEVERAGE CHANGE ŌĆö The single change that would have the biggest positive impact.

Return ONLY valid JSON in this exact format:
{
  "overall_score": number (1-10),
  "categories": [
    {
      "name": string,
      "score": number (1-10),
      "assessment": string (2-3 sentences, specific and honest),
      "highest_leverage_action": string (1 sentence, actionable)
    }
  ],
  "strongest_feature": string,
  "highest_leverage_change": string,
  "notes": string (any additional observations)
}
```

**deepDivePrompt(category, score)** ŌĆö interpolate `[SCORE]` and `[CATEGORY]`:

```
You are a research expert in male physical optimization. The user scored [SCORE]/10 in [CATEGORY].

Do a comprehensive deep dive using your training knowledge. Draw from:
- Peer-reviewed research (PubMed, NIH studies)
- Evidence-based practitioners (dermatologists, orthotropists, sports scientists)
- Books that are genuinely respected in this field (not pseudoscience)
- High-credibility YouTube channels and communities (e.g. r/SkincareAddiction, r/mewing, r/fitness)
- Specific product and routine recommendations with reasons

Structure your response as:
1. WHY this score ŌĆö what the science says about the underlying cause
2. WHAT actually works ŌĆö interventions with evidence behind them (and what doesn't work despite hype)
3. TIMELINE ŌĆö realistic improvement expectations
4. RESOURCES ŌĆö specific books, channels, tools (with names, not vague references)
5. 30-DAY STARTER PLAN ŌĆö exactly what to do in the next 30 days

Be direct. If something takes years (e.g. mewing), say so. If surgery is the only real fix for something, say that too. Do not sugarcoat. The user wants truth, not comfort.
```

**roadmapPrompt(analysisJSON, profile)** ŌĆö interpolate `[JSON]` and the
profile fields:

```
Based on this full facial assessment: [JSON], and this user profile: age [X], height [X], weight [X], sleep [X]hrs, training [X]x/week, diet [X], beard [yes/no], skincare [yes/no]:

Build a realistic 90-day optimization roadmap. Priority order improvements by leverage (biggest ROI first).

Structure:
- WEEK 1-2: Foundation (habits to start immediately, zero-cost or low-cost)
- WEEK 3-8: Build phase (consistency, adding tools/products/routines)
- WEEK 9-12: Refine (advanced adjustments, assess progress)

Cover: grooming, skincare, training adjustments relevant to facial goals (e.g. cutting body fat to reveal jaw), sleep optimization, hydration, nutrition flags, style/haircut notes.

Format as structured JSON with phases, weeks, and daily/weekly tasks.
Be specific. "Drink more water" is not acceptable. "Drink 3.5L of water daily, front-load in morning to aid lymphatic drainage" is acceptable.
```
